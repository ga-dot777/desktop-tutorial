2. Priebežný test - variant A¶

Poznámka

  • riešenia úloh ulož do jedného súboru a odovzdaj na testovací server https://vektor.fmph.uniba.sk/

    • prvé tri riadky súboru s riešením musia obsahovať:

      # 2. test variant A
# student: Janko Hrasko
# datum: 15.12.2025

      zrejme ako študenta uvedieš svoje meno.

  • nepoužívaj global ani import

  • svojim funkciám (oproti zadaniu) môžeš zmeniť mená parametrov, môžeš si zadefinovať pomocné funkcie


  1. Prvé štyri riadky Pascalovho trojuholnika zvykneme vypisovať takto:

          1
    1   1
  1   2   1
1   3   3   1

    Napíš funkciu pascal(n), ktorá zostrojí a vráti dvojrozmernú tabuľku (zoznam zoznamov) prvých n riadkov Pascalovho trojuholníka. Táto dvojrozmerná tabuľka bude obsahovať Pascalov trojuholník otočený, tak že 0-riadok bude v ľavom hornom rohu. Každý ďalší riadok sa nachadza šikmo pod predchádzajúcim. Napríklad:

    >>> print(*pascal(3), sep='\n')
    [1, 1, 1, 1]
    [1, 2, 3]
    [1, 3]
    [1]

  1. V textovom súbore sú slová oddelené medzerami, prípadne koncami riadkov. Napíš funkciu subor(meno_suboru, typ), ktorá spracuje tieto slová a z niektorých z nich vytvorí výsledný zoznam (funkcia vráti tento zoznam ako výsledok):

    • ak je parameter typ znakový reťazec, do výsledného zoznamu pridá všetky slová začínajúce týmto reťazcom

    • inak, je tento parameter pythonovským typom (napríklad int), do výsledného zoznamu vtedy pridá všetky prekonvertované slová týmto typom (ak sa to dá), napríklad všetky celé čísla

    Zadefinuj:

    def subor(meno_suboru, typ):
    ...

  1. Dodefinuj metódy triedy:

    class Cisla:
    ...

    tak, aby fungovalo:

    >>> Cisla(5, 3, 4)    # parametrami je ľubovoľný nenulovy počet celých čísel
    12
>>> c = Cisla(5, 3, 4, 2)
>>> c.max()
    5
>>> c.sucin()
    120
>>> c                 # __repr__ vráti súčet
    14
>>> c.pridaj(7)       # do svojho zoznamu si pridá ďalšie číslo
>>> c.max()

  1. Vygeneruj všetky súčty so štyrmi sčítancami s celými číslami zo zadanej postupnosti v tvare znakových reťazcov '2+5+1+2'. Všetky tieto výrazy dávajú rôzne súčty, niektoré súčty sa opakujú. Napríklad, výrazy '2+5+1+2' a '3+2+3+2' dávajú rovnaký súčet. Na poradí sčítancov záleží a preto '2+5+1+2' a '2+1+5+2' sú rôzne výrazy s rovnakým súčtom. Napíš funkciu sucty(*postupnost), ktorá vyrobí množinu všetkých takých reťazcov, ktoré dávajú najčastejší súčet. Funkcia vráti množinu, napríklad:

    >>> sucty(1, 2, 3, 5)
    {'2+5+1+2', '2+1+5+2', '3+2+3+2', ...}   # toto nemusí byť správny výsledok

  1. Napíš funkciu zoskup(funkcia, postupnost), ktorá pre každý prvok postupnosti aplikuje danú funkciu a podľa týchto hodnôt (budeme ich volať kľúče) všetky prvky zaradí do skupín. Funkcia vráti n-ticu skupín, pričom každá skupina je opäť n-tica, v ktorej prvý prvok je kľúč skupiny a druhý je n-tica prvkov skupiny (v rovnakom poradí, ako boli v pôvodnej postupnosti). Napríklad:

    >>> zoskup(lambda x: x%2, (3, 7, -4, 1, 2))
    ((1, (3, 7, 1)), (0, (-4, 2)))

  1. Zadefinuj takúto mapovaciu funkciu filter_2(funkcia, postupnost), ktorá aplikuje danú funkciu (prvý parameter) na všetky prvky nejakej postupnosti a do výsledného zoznamu vloží len tie, pre ktoré funkcia nevráti False. Ak je niektorý prvok v postupnosti zoznam alebo n-tica, aplikuje toto mapovanie aj na jeho prvky. Zrejme aj tieto podzoznamy môžu obsahovať ďalšie podzoznamy, na ktoré bude treba aplikovať danú funkciu. Ak takýto vnorený zoznam po aplikovaní funkcie by ostal prázdny, tak ho do výsledku nezaradí. Napríklad:

    >>> filter_2(lambda x: x%2, (3, 7, (-4, 6), 1, 2))
    [3, 7, 1]

  1. Preprogramuj:

    def rob(x, y, n, d):
    if n > 0:
        if n % 2:
            u = 1
        else:
            u = -1
        canvas.create_line(x, y, x - d * u, y - d)
        rob(x - d * u, y - d, n - 1, d * .7)
        canvas.create_line(x, y, x + d * u, y - d)
        rob(x + d * u, y - d, n - 1, d * .5)
    else:
        canvas.create_oval(x - 2, y - 2, x + 2, y + 2)

    tak, aby funkcia rob, ale bez kreslenia, vygenerovala (pomocou yield) postupnosť súradníc (teda tuple dvoch čísel) stredov malých kružníc. Napríklad, volanie:

    li = [*rob(200, 250, 5, 70)]

    by malo vrátiť zoznam dvojíc všetkých súradníc v správnom poradí.
